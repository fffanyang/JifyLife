## JifyLife 纯前端 PWA 改造

将 JifyLife 从前后端分离架构改造为**纯前端 PWA**，不再需要任何服务器，数据存在手机浏览器本地 IndexedDB 中。

- `apps/web/src/services/db.ts` — 基于 Dexie.js 的 IndexedDB 数据库定义

### 改造的文件
- `apps/web/src/pages/MainView.tsx` — Mock 数据 → IndexedDB 读写
- `apps/web/src/App.tsx` — 移除 React Query、登录路由
- `apps/web/src/components/TopBar/TopBar.tsx` — 移除认证相关逻辑
- `apps/web/vite.config.ts` — 移除 API proxy，增强 PWA 离线缓存

- `apps/web/src/services/authService.ts` — 弃用

## 全面重构：马伯庸式人生日记

用户要求重新调整需求，将 JifyLife 彻底改造为"马伯庸式人生日记"——一个极简本地日记 PWA。

### 架构

- React + Vite + TailwindCSS v4 + react-router-dom + zustand

- 全部旧服务（api.ts, authService.ts, entryService.ts, localEntryService.ts）
- 全部旧 store（authStore.ts, entryStore.ts, settingsStore.ts）
- 旧页面（LoginPage.tsx, MainView.tsx）
- 旧 hook（useMediaQuery.ts）
- 移除依赖：@jifylife/shared, uuid, @types/uuid

### 新建的文件
- `src/types/diary.ts` — 日记数据模型 + 字段/天气常量
- `src/services/db.ts` — Dexie.js IndexedDB 定义（MaBoyongDiary 库）
- `src/services/diaryService.ts` — 完整 CRUD + Markdown 生成 + 统计 + 导出
- `src/stores/diaryStore.ts` — Zustand 日记状态管理
- `src/pages/HomePage.tsx` — 首页（日记列表 + 今日快捷入口）

- `src/pages/DiaryDetailPage.tsx` — 日记详情页（马伯庸风格排版）
- `src/pages/ReviewPage.tsx` — 回顾页（周/月/年统计卡片）
- `src/pages/SettingsPage.tsx` — 设置页（存储路径 + 导出 + 版本）

### 4 个主页面
1. 编辑页：日期前后翻页 + 天气选择 + 事项/阅读/输入/输出/生活 5 个文本框 + 作息（起床/睡觉时间选择器）+ 图片（1-9张）+ 保存
2. 日记详情页：分模块排版展示，图片加载失败显示"图片已删除"
3. 回顾页：周/月/年切换 + 作息/事务/生活高频统计卡片 + 一句话总结 + 品牌水印（适合截图分享）
4. 设置页：存储路径 + 复制 + 导出全部日记（.md + index.json）+ 清除数据 + 版本信息

## GitHub Pages 部署配置

配置项目支持通过 GitHub Pages 部署为纯静态 PWA 站点。

### 修改的文件
- `apps/web/vite.config.ts` — 添加动态 `base` 路径（通过 `VITE_BASE_PATH` 环境变量），PWA manifest 的 `start_url` 和 `scope` 改为相对路径 `"."`，图标路径改为相对路径
- `apps/web/src/App.tsx` — `BrowserRouter` 改为 `HashRouter`（GitHub Pages 不支持 SPA 路由重写）
- `README.md` — 添加 GitHub Pages 部署步骤说明

### 新增文件
- `.github/workflows/deploy.yml` — GitHub Actions 自动部署工作流：checkout → pnpm install → build（带 VITE_BASE_PATH）→ upload-pages-artifact → deploy-pages

### 部署步骤
1. 创建 GitHub 仓库并推送代码
2. Settings → Pages → Source 选 "GitHub Actions"
3. 推送到 main 分支自动部署
4. 访问 `https://<username>.github.io/JifyLife/`
5. 手机浏览器打开后"添加到主屏幕"安装为 PWA

### 构建验证
- 构建成功，所有资源路径正确带上 `/JifyLife/` 前缀
- manifest.webmanifest 使用相对路径，PWA 安装不受 base path 影响

## Cloudflare Pages 部署

由于 GitHub Pages（github.io）在国内访问不稳定，用户选择同时使用 Cloudflare Pages 作为备用/主力部署平台。

### 代码层面
- 不需要修改任何代码，Vite 配置的 `VITE_BASE_PATH` 默认为 `/`，Cloudflare Pages 部署在根路径正好匹配
- HashRouter 和相对路径 PWA manifest 同样兼容根路径部署
- Cloudflare Pages 在云端构建，配置通过网页 UI 填写

### Cloudflare Pages 构建配置
- Framework preset: None
- Build command: `npx pnpm install --frozen-lockfile && npx pnpm build:web`
- Build output directory: `apps/web/dist`
- Root directory: `/`（默认）
- Node.js version 环境变量: `NODE_VERSION=20`

### 部署地址
- Cloudflare Pages: `https://jifylife.pages.dev`（或用户自选的项目名）
- GitHub Pages: `https://fffanyang.github.io/JifyLife/`（备用）
